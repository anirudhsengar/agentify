# Draft cost accounting

Budget-controlled draft runs use the static `pricing_policy` in `.github/agentify-shadow.json`. Every entry identifies an exact provider, model, effective policy version, and input, cached-input (when applicable), and output USD-per-million-token rates. Pricing is never fetched during a run. Provider pricing changes over time, so maintainers must review and update this versioned configuration; stale pricing can understate exposure. An unknown provider/model/version fails closed and is never assigned zero cost.

Before every provider request, Agentify reserves the configured maximum input tokens plus the model's enforced maximum output tokens at the matching rates. A request is rejected before dispatch if that conservative reservation exceeds the remaining engagement budget. After a response, measured provider token usage replaces the reservation and unused exposure is released. Each record includes provider, model, input/cached-input/output/reasoning/total tokens when supplied, safe provider request ID when available, timestamps, calculated cost, reservation, pricing-policy version, and `measured`, `estimated`, or `unavailable` status.

`budget_usd` is the configured maximum, not spend. `reserved_cost_usd` is outstanding worst-case exposure. `measured_cost_usd` is calculated from provider-reported usage. `estimated_cost_usd` uses the unreconciled conservative reservation when usage is missing. `remaining_budget_usd` subtracts measured, estimated, and outstanding reserved exposure. A measured-required policy is non-publishable when any call lacks measured usage; it never claims an exact cost.

The guarantee is admission-based: no new call begins when its reserved maximum exceeds the remaining budget; every call has a bounded output limit; cumulative measured, estimated, and reserved exposure is persisted; and execution stops before another call after exhaustion. A remote provider may still charge for one in-flight request after local cancellation. Agentify records that possible exposure and does not promise zero overshoot unless the provider contract acknowledges cancellation before billing.
